# 프로세스
프로세스는 **실행 중인 프로그램**을 의미하며 **CPU가 처리하는 작업의 기본 단위**이다. 프로세스는 보조기억장치에 영구적으로 저장되어 있다가 프로그램이 실행되었을 때, 주기억장치에 할당된다.

현대 컴퓨터에서는 여러 프로세스가 할당되고 종료된다. 이 특성으로 인해 여러 프로세스를 할당해야 할 때, 어느 순서로 할당을 해야할지 결정해야 하는데 이를 **프로세스 스케줄링**이라 한다. (스케줄링에 대한 것은 <스케줄링> 파트에서 자세히 설명)

이 말고도 프로세스는 할당되었을 때, 주기억장치에서 독립적인 메모리 공간을 소유하는데 이 안에는 스택, 힙, 코드, BSS, 데이터 영역 등이 존재한다.

## 프로세스 상태
프로세스는 생성되어 종료될 때까지 아래와 같은 변화를 거친다.

- **New (생성)**: 프로세스가 생성되기 위해 준비된 상태로 아직 보조기억장치에 존재한다.

- **Ready (준비)**: 메모리(주기억장치)에 적재되어 CPU의 할당을 기다리는 상태로 프로세스 제어 블록(PCB)이 준비큐에 연결된다.
	- **프로세스 제어 블록 (PCB)**: 프로세스의 중요한 정보들을 담고 있는 블록으로, 중간에 멈춘 프로세스를 다시 할당할 때 PCB의 정보를 이용하여 다시 이어서 작업을 할 수 있게 해준다. 
	- **준비큐**: 준비 상태의 프로세스들을 모아둔 큐(Queue)

- **Running (실행)**: CPU를 할당받아 실제로 실행되고 있는 상태

- **Waiting (대기)**: 특정 입출력이나 다른 이벤트로 인해 프로그램이 해당 이벤트를 완료를 기다리고 있는 상태

- **Terminated (종료)**: 프로세스의 실행이 완료되고 해당 프로세스의 PCB가 삭제된 상태


> **PCB에 저장되는 대표적인 정보**
> 1. 프로세스 식별자 (pid)
> 2. 프로세스 상태 (ex. New, Ready...)
> 3. 프로그램 카운터 (PC)
> 4. CPU 레지스터 및 일반 레지스터
> 5. CPU 스케줄링 정보
> 6. 메모리 관리 정보 (해당 프로세스의 주소 공간 등)
> 7. 프로세스 계정 정보 (페이지 테이블, 소유자 등)
> 8. 입출력 상태
> 9. 포인터 (프로세스가 위치한 포인터, 부모 프로세스에 대한 포인터 등)
